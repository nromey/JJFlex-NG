## What needs Track A (real radio)

**TLS path verification:**
- Watch for the Debug log line `SslClientTls12 negotiated protocol: Tls13` (or `Tls12` fallback) on first connect. If it logs `None` or no log at all, the wrapper's not in the path.
- Verify a normal LAN connect still succeeds with the new `if (IsWan || PublicTlsPort > 0)` selector. On firmware that exposes `PublicTlsPort`, this shifts LAN to the TLS path; on older firmware, no change.
- SmartLink / WAN connect: same protocol log, but on the WAN path. Verify the new `Connect()` async-then-`.GetAwaiter().GetResult()` flow doesn't hang the UI thread on slow networks.

**Discovery race verification:**
- Trigger `apiInit(force=true)` (network adapter toggle, or a Refresh in the radio chooser) while a prior Receive task is still pending. The Output window should show:
  - `Discovery.Receive: task started (udp=set)` (new task)
  - `Discovery.Receive: task exited cleanly` (old task exiting)
  - No NRE.
- `Discovery.Receive: exiting immediately, udp was null at entry` is fine if it appears occasionally — that's the null-guard catching a degenerate startup ordering.

**Vita pipeline (audio):**
- v4.2.18 added `HighPriorityTaskScheduler`, `MmcssPipelineScheduler`, and a `VitaPacketBase` refactor. JJFlex doesn't consume any of these directly, but vendor code calls them under the hood for RX/TX/DAX paths. Smoke RX, TX (CW/SSB), DAX RX, DAX TX, DAX MIC. Listen for dropouts or scheduling stalls.

**AS-retry pathway (`project_as_retry_pathway_regression.md`):**
- Note whether the secondary AS retry + reconnect feels better, worse, or the same in v4.2.18. The Sprint 26+27 networking overhaul made it jankier; v4.2.18 may shift that one direction or another.

**HAAPI (external amp, if Noel has one connected):**
- The HAAPI event surface was rewritten (`AmplifierFault` → `HaapiFault(noun, reason)`, plus `HaapiWarning`/`HaapiWarningCleared` and per-meter data-ready events). JJFlex doesn't subscribe to any of this today, so behavior should be unchanged. A single connect with the amp powered on, just to confirm no startup crash, is enough.

**Feature-license flags:**
- v4.2.18 adds `LicenseFeatNAVTEX`, `LicenseFeatRapidmLogging`, `LicenseFeatPanadapterVisuals`, `LicenseFeatFilterPresetConf`. JJFlex doesn't gate any UI on these yet, so just confirm `FeatureLicense` parsing doesn't blow up on a real license string from a Plus radio.
