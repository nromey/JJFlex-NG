# Phase 7 — No-radio smoke check

## Outcome: deferred to Track A

A meaningful smoke check requires either sighted observation or a screen reader to verify the "no radio connected" announcement, the main window contents, and any startup dialog state. Track B is an autonomous, headless agent with neither — launching `JJFlexRadio.exe` here would either:

- Show a window I can't actually inspect, then sit on the screen until something interacts with it, or
- Pop a modal (stuck-on-connecting, no-radio guard, license-renewal prompt) that I'd need to dismiss without being able to perceive it.

The autonomous correct call per TRACK-INSTRUCTIONS — "Don't kill the app via taskkill — that's the stuck-modal pattern from `project_dialog_escape_rule.md`" — is to not start a process I can't clean up gracefully.
